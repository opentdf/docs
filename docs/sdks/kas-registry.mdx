# KAS Registry

:::note
The methods on this page are provided through the [platform service client](/sdks/platform-client).
:::

The **KAS Registry** tracks the Key Access Servers (KAS) that your platform trusts for key wrapping and unwrapping operations. Each registered KAS entry holds a URI, a name, and one or more public keys. The registry also manages the lifecycle of those keys — including rotation and base key assignment.

:::note Hard-delete
KAS registry entries use hard delete (not deactivation). Deleted entries cannot be restored.
:::

```go
package main

import (
	"context"
	"log"

	"github.com/opentdf/platform/protocol/go/policy/kasregistry"
	"github.com/opentdf/platform/sdk"
)

	client, err := sdk.New("http://localhost:8080",
		sdk.WithClientCredentials("opentdf", "secret", nil),

	}
```

### Get a Key Access Server

```go
package main

import (
	"context"
	"log"

	"github.com/opentdf/platform/protocol/go/policy/kasregistry"
	"github.com/opentdf/platform/sdk"
)

	resp, err := client.KeyAccessServerRegistry.GetKeyAccessServer(context.Background(),
		&kasregistry.GetKeyAccessServerRequest{

	log.Printf("KAS: %s\n  URI: %s\n  ID:  %s\n", kas.GetName(), kas.GetUri(), kas.GetId())

		&kasregistry.GetKeyAccessServerRequest{
			Identifier: &kasregistry.GetKeyAccessServerRequest_Uri{
				Uri: "https://kas.example.com",
			},
		},
	)
	if err != nil {
		log.Fatal(err)
	}
	log.Printf("KAS by URI: %s\n", respByUri.GetKeyAccessServer().GetName())
}
```

## KAS Keys

Each KAS entry holds one or more cryptographic keys. Keys have a lifecycle: they are created, optionally set as the **base key** (the platform default), and eventually **rotated** out when replaced by a newer key. A rotated key retains its status (`KEY_STATUS_ROTATED`) so that TDFs encrypted under it can still be decrypted.

**Key modes** determine how the private key material is stored and used:

| Mode | Description |
|---|---|
| `KEY_MODE_CONFIG_ROOT_KEY` | Private key is wrapped by a local config-managed KEK |
| `KEY_MODE_PROVIDER_ROOT_KEY` | An external provider manages the KEK |
| `KEY_MODE_REMOTE` | All crypto operations are delegated to a remote KMS/HSM |
| `KEY_MODE_PUBLIC_KEY_ONLY` | Only the public key is registered; used for wrapping by external clients |

**Supported algorithms:** `ALGORITHM_RSA_2048`, `ALGORITHM_RSA_4096`, `ALGORITHM_EC_P256`, `ALGORITHM_EC_P384`, `ALGORITHM_EC_P521`

```java
package io.opentdf.platform;

import io.opentdf.platform.policy.kasregistry.*;

public class CreateKasKeyExample {
    public static void main(String[] args) throws Exception {

            .build();

            .createKeyBlocking(req, Collections.emptyMap()).execute();
        System.out.println("Created key ID: " + resp.getKasKey().getId()

        sdk.close();
```

### Rotate a Key

Rotation replaces the active key on a KAS with a new one. The platform re-maps all existing attribute, namespace, and value grants that referenced the old key to the new key. The old key transitions to `KEY_STATUS_ROTATED` and remains available for decrypting TDFs that were encrypted under it.

<details id="rotate-key">
<summary>Rotate a Key</summary>
<Tabs>
<TabItem value="go" label="Go">

```go
package main

import (
	"context"
	"log"

	"github.com/opentdf/platform/protocol/go/policy"
	"github.com/opentdf/platform/protocol/go/policy/kasregistry"
	"github.com/opentdf/platform/sdk"
)

func main() {
	client, err := sdk.New("http://localhost:8080",
		sdk.WithClientCredentials("opentdf", "secret", nil),
	)
	if err != nil {
		log.Fatal(err)
	}

	resp, err := client.KeyAccessServerRegistry.RotateKey(context.Background(),
		&kasregistry.RotateKeyRequest{
			// Identify the key being rotated out
			ActiveKey: &kasregistry.RotateKeyRequest_Id{
				Id: "key-uuid-0000-0000-0000-000000000001",
			},
			// Provide the replacement key
			NewKey: &kasregistry.RotateKeyRequest_NewKey{
				KeyId:     "my-rsa-key-v2",
				Algorithm: policy.Algorithm_ALGORITHM_RSA_2048,
				KeyMode:   policy.KeyMode_KEY_MODE_CONFIG_ROOT_KEY,
				PublicKeyCtx: &policy.PublicKeyCtx{
					Pem: "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----",
				},
			},
		},
	)
	if err != nil {
		log.Fatal(err)
	}

	newKey := resp.GetKasKey()
	rotated := resp.GetRotatedResources()
	log.Printf("New active key kid: %s\n", newKey.GetKid())
	log.Printf("Rotated out key kid: %s\n", rotated.GetRotatedOutKey().GetKid())
	log.Printf("Re-mapped attribute definitions: %d\n", len(rotated.GetAttributeDefinitionMappings()))
	log.Printf("Re-mapped attribute values:      %d\n", len(rotated.GetAttributeValueMappings()))
	log.Printf("Re-mapped namespaces:            %d\n", len(rotated.GetNamespaceMappings()))
}
```

</TabItem>

## Base Key

The **base key** is the platform's default KAS key — used when no explicit key grant is specified for an attribute or namespace. Only one key can be the base key at a time.

### Get the Base Key

<details id="get-base-key">
<summary>Get the Base Key</summary>
<Tabs>
<TabItem value="go" label="Go">

```go

	"context"
	"log"

	"github.com/opentdf/platform/sdk"
)

	client, err := sdk.New("http://localhost:8080",
		sdk.WithClientCredentials("opentdf", "secret", nil),
	)
	if err != nil {
		log.Fatal(err)
	}

	resp, err := client.KeyAccessServerRegistry.GetBaseKey(context.Background(),
		&kasregistry.GetBaseKeyRequest{},

		base.GetKid(), base.GetKasUri(), base.GetAlgorithm())
```

## Key Mappings

Key mappings show which namespaces, attribute definitions, and attribute values have grants tied to a specific key. Use this before rotating or deleting a key to understand the blast radius.

<details id="list-key-mappings">
<summary>List Key Mappings</summary>
<Tabs>
<TabItem value="go" label="Go">

```go
package main

import (
	"context"
	"log"

	"github.com/opentdf/platform/protocol/go/policy/kasregistry"
	"github.com/opentdf/platform/sdk"
)

func main() {
	client, err := sdk.New("http://localhost:8080",
		sdk.WithClientCredentials("opentdf", "secret", nil),
	)
	if err != nil {
		log.Fatal(err)
	}

	resp, err := client.KeyAccessServerRegistry.ListKeyMappings(context.Background(),
		&kasregistry.ListKeyMappingsRequest{
			Identifier: &kasregistry.ListKeyMappingsRequest_Id{
				Id: "key-uuid-0000-0000-0000-000000000001",
			},
		},
	)
	if err != nil {
		log.Fatal(err)
	}
	for _, m := range resp.GetKeyMappings() {
		log.Printf("Key kid: %s (KAS: %s)\n", m.GetKid(), m.GetKasUri())
		log.Printf("  Namespace grants:         %d\n", len(m.GetNamespaceMappings()))
		log.Printf("  Attribute grants:         %d\n", len(m.GetAttributeMappings()))
		log.Printf("  Attribute value grants:   %d\n", len(m.GetValueMappings()))
	}
}
```

Omit `Identifier` to list mappings for all keys.
